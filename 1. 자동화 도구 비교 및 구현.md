## **기본 단어와 개념**

| 단어 | 개념 |
| --- | --- |
| make, zapier | 반복적인 작업을 자동화할 수 있게 만드는 노코드 자동화 워크플로우 |
| trigger | 자동화 작업이 시작되는 이벤트나 조건 | 
| action | 자동화 작업 과정에서의 행동 단위 |
| 'fileter' or 'if/else' | 받은 데이터의 경로를 결정하는 조건 관문
| router | 자동화로 설정된 동작이 흐르는 경로 |

## **자동화 도구 비교분석**
(민감정보는 노출X)

*사용 도구: make, zapier
*구현: 구글 폼에서 설문을 받으면 설문 내에서 선택한 항목에 맞춰 디스코드 내 각각의 방에 알림이 가도록 설정
*조건분기: if 구글 폼 내 1번 항목 선택 후 제출, if 구글 폼 내 2번 항목 선택 후 제출, if 구글 폼 내 3번 항목 선택 후 제출
*구현 과정 (각각 스크린샷 필수)
 : 1. 디스코드 테스트방 생성
   2. 채널 내에 "미션 테스트 1번 문항 선택"의 이름으로 번수만 바꿔 3개의 채널 생성
   3. Google Forms에서 테스트용 폼을 생성하여 항목을 3개 지정 후 폼 생성
   4. make, zapier에서 각각 Google Forms을 트리거로 설정 
   5. make에서는 router를, zapier에서는 path를 추가하여 Google Forms의 체크 항목인 1, 2, 3번 항목에 맞춰 각각
      항목 1번 체크 후 제출 시 "미션 테스트 1번 문항 선택" 디스코드 채널에 "1번 문항을 선택한 폼이 접수되었습니다"이라는 알람을 보내도록하며 같은 방법으로 조건분기(필터) 3개 생성
   6. 각각 조건분기 끝에 디스코드 채널방으로 알림이 가도록 설정
*도구별 완성된 워크플로우 구성 화면 캡쳐
*실행결과: (조건 분기 별 스크린샷도 필수)
*비교:

   *최소 5개 이상의 비교 항목(UX/UI, 설정 난이도, 연동 서비스 범위, 무료 플랜 범위, 실행 로그 확인 방식 등)
   *각 도구의 장단점 정리, 어떤 상황에서 적합한지 의견 작성

3. 계정/연동 환경 리스크 완화 가이드 (구글 시트 + 지메일 + 웹훅 or 구글 시트 + 슬랙or디스코드 + 웹혹)
