## **기본 단어와 개념**

| 단어 | 개념 |
| --- | --- |
| make, zapier | 반복적인 작업을 자동화할 수 있게 만드는 노코드 자동화 워크플로우 |
| trigger | 자동화 작업이 시작되는 이벤트나 조건 | 
| action | 자동화 작업 과정에서의 행동 단위 |
| 'fileter' or 'if/else' | 받은 데이터의 경로를 결정하는 조건 관문
| router | 자동화로 설정된 동작이 흐르는 경로 |

## **자동화 도구 비교분석**
(민감정보는 노출X)

*구현 과정 (각각 스크린샷 필수)
 : 
*도구별 완성된 워크플로우 구성 화면 캡쳐
*실행결과: (조건 분기 별 스크린샷도 필수)
*비교: 비교 축 _ 설정 난이도, 무료 플랜 범용성, 연동 서비스 범위, 실행 로그 확인 방식, UX/UI
      각 도구 장점/단점, 나에게 관리권한이 없는 외부 사이트나 앱의 트리거를 감지하고 액션을 만드는 것은 make가 더 편리함 (또 다른 이유도 만들기)
*과금 리스크 완화법 가이드
