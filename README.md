make나 zapier같은 플랫폼을 활용하여 코딩없이 trigger(시작 이벤트)와 action(처리 동작)을 시각적으로 연결해 반복 업무 자동화
워크플로우를 서로 다른 2개 이상의 자동화 도구로 구현 + 비교 분석 후 자유주제로 나만의 자동화 파이프라인 설계 및 구현

*실제로 동작하는 자동화 워크플로우 구현
*트리거 1개 이상 포함
*액션 1개 이상 포함
*조건 분기(필터/라우터) 1개 이상 포함
(조건 분기 구조가 포함된 경우, 각 분기 경로가 실제로 1회 이상 실행된 결과를 확인할 수 있어야 함)

1. 자동화 도구 비교 구현
- 동일한 자동화 워크플로우를 2개 이상의 도구로 구현
- 각 도구별 워크플로우 구성 화면 캡쳐
- 실행 결과 화면 캡쳐
- 비교 분석 보고서 (markdown 또는 PDF)
+ 서로 다른 2개 이상의 자동화 도구 사용
+ 동일한 워크플로우 구조로 구현
+ 비교 분석 보고서 내에는 사용한 도구명, 구현 과정 요약,]
                     최소 5개 이상의 비교 항목(UX/UI, 설정 난이도, 연동 서비스 범위, 무료 플랜 범위, 실행 로그 확인 방식 등),
                     각 도구의 장단점 정리, 어떤 상황에서 적합한지 의견 작성

2. 자유 주제 자동화 설계 및 구현
- 자동화할 반복 업무 1개 정의
- 자동화 도구 1개 선정 및 선정 이유 작성
- 워크플로우 설계 문서 (설명 또는 다이어그램 포함)
- 구현 화면 캡쳐
- 실행 결과 화면 캡쳐
+ 자동화할 반복 업무 1개 정의
+ 도구 1개 선정 및 선정 이유 작성
+ 자동 실행 구조 구현
+ 워크플로우 흐름 설명 포함
* 트리거 발생 시, 자동 실행되어야 함
* 자동화 툴을 비교분석하기위해 도구를 반
